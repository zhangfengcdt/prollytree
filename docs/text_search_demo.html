Text Indexing & Vector Search

ProllyTree's NamespacedKvStore can host versioned approximate-nearest-neighbour indexes inside any namespace, alongside the primary key-value tree. Commits, branches, and three-way merges atomically cover both the data and every owned sub-index. This page is a self-contained illustration of the user-facing surface — no install required.

How it fits together

A NamespacedKvStore holds many independent prolly trees, one per namespace, plus zero or more text sub-indexes per namespace. The primary tree is the source of truth (doc id → body bytes); each text index stores only (id, vector) pairs and gives you top-k similarity search.

Cascade: one write, both trees

Toggle cascade on, then "insert" a new document. With cascade on, a single ns_insert writes to both the primary tree and every registered text index inside the same commit. With cascade off the index doesn't see the write — drift accumulates, surfaceable later via audit_text_index.

The Python API

Real code — copy/paste runnable once you've pip install prollytree.

from prollytree import NamespacedKvStore, MiniLmEmbedder

store = NamespacedKvStore("./store")
emb   = MiniLmEmbedder()                  # bundled Candle + all-MiniLM-L6-v2

# Creates the index on first call; on reopen, validates that the supplied
# embedder's id + version match what's persisted on disk.
store.text_index_open("docs", "by_body", emb)
# Primary tree carries the source bytes; the index carries the vectors.
# Without the primary write, search hits can't be resolved back to text.
docs = {
    b"doc:1": "the quick brown fox",
    b"doc:2": "lazy dog asleep on the mat",
}
for doc_id, text in docs.items():
    store.ns_insert("docs", doc_id, text.encode())                # primary
    store.text_index_insert("docs", "by_body", doc_id, text)  # index
store.commit("seed corpus + index")
# Opt in once — then `ns_insert` and `ns_delete` auto-mirror into the listed
# indexes. Runtime-only, not persisted.
store.set_cascade("docs", ["by_body"])

store.ns_insert("docs", b"doc:3", b"prolly trees are great")
store.commit("cascade-driven indexing")        # index updated too

store.cascade_for_namespace("docs")        # -> ['by_body']
store.clear_cascade("docs")                  # opt out
# Returns (id_bytes, distance) tuples. Lower distance = closer.
hits = store.text_index_search("docs", "by_body", "vulpine animal", k=5)

for doc_id, score in hits:
    body = store.ns_get("docs", doc_id).decode()     # resolve back
    print(f"{doc_id} (d={score:.3f}): {body}")
# Split each document into many chunks at indexing time. Search dedups
# results back to documents at their best chunk's score.
store.text_index_open("logs", "by_line", emb, chunker="line")
store.text_index_insert("logs", "by_line", b"log:42",
                        "alpha\nbeta\ngamma")              # 3 chunks

store.text_index_len("logs", "by_line")            # 1 doc
store.text_index_chunk_count("logs", "by_line")    # 3 chunks
# Wrap any Python embedding function as an Embedder — OpenAI, Cohere,
# sentence-transformers, your own pipeline.
from prollytree import CallableEmbedder
from openai import OpenAI

client = OpenAI()
def openai_embed(text):
    return client.embeddings.create(
        input=text, model="text-embedding-3-small"
    ).data[0].embedding

emb = CallableEmbedder(
    id="openai:text-embedding-3-small",
    version="2024-01",
    dim=1536,
    embed_fn=openai_embed,
)
